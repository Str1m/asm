## Условие tl;dr
Даны два числа:
- M — начальное число
- N — сколько шагов сделать
Один шаг:
1. развернуть цифры числа в десятичной записи
2. прибавить развёрнутое число к исходному
Нужно проверить:
- получится ли **ровно после** **N** **шагов** палиндром
Если да:
- вывести Yes
- на следующей строке получившийся палиндром
Иначе:
- вывести No
Важно:
- если палиндром появился раньше, но после N шагов число уже не палиндром, ответ всё равно No
- массивы использовать нельзя
## Решение — псевдокод
Идея:
- сделать функцию reverse_num(x) — переворот цифр
- сделать функцию is_palindrome(x) — проверить, равен ли x своему перевороту
- затем выполнить ровно N шагов
- в конце проверить, палиндром ли получившееся число
``` rust
fn reverse_num(x: u32) -> u32 {
    let a = x;
    let rev = 0;

    while a > 0 {
        rev = rev * 10 + (a % 10);
        a /= 10;
    }

    return rev;
}

fn is_palindrome(x: u32) -> bool {
    return x == reverse_num(x);
}

fn solve() {
    read m, n;

    let mut x = m;

    // Делаем ровно n шагов
    for i in 0..n {
        let rev = reverse_num(x);
        x = x + rev;
    }

    // Проверяем именно результат после n шагов
    if is_palindrome(x) {
        print "Yes";
        print newline;
        print x;
    } else {
        print "No";
    }
}
```
