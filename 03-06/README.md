# 03-06. 
## Условие tl;dr
Дано:
- N — количество чисел в последовательности (N ≤ 1000)
- затем N беззнаковых 32-битных чисел
- затем число k (k ≤ 32)
Для каждого числа нужно посчитать, сколько нулей в его двоичной записи являются значащими, то есть находятся между старшим и младшим единичными битами.

Нужно вывести, сколько чисел в последовательности имеют ровно k значащих нулей.
Требование:
- функцию подсчёта значащих нулей нужно оформить отдельно, по cdecl.
## Решение — псевдокод
Идея:
1. найти младший установленный бит
2. сдвинуть число вправо до него
3. дальше идти по битам, пока число не станет равно 1
    - если текущий бит 0, увеличиваем счётчик
    - если 1, просто идём дальше
    
Так мы считаем нули между крайними единицами, а не ведущие и не хвостовые.
``` rust
fn significant_zeros(x: u32) -> u32 {
    if x == 0 {
        return 0;
    }

    let mut a = x;

    while (a & 1) == 0 {
        a >>= 1;
    }

    let mut count = 0;

    while a > 1 {
        a >>= 1;
        if a > 0 && (a & 1) == 0 {
            count += 1;
        }
    }

    return count;
}

fn solve() {
    read n;

    let mut arr = sequence of n numbers
    read k

    let mut ans = 0
    for x in arr {
        if significant_zeros(x) == k {
            ans += 1
        }
    }

    print ans
}
```
